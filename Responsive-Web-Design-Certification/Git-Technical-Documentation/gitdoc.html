What is Git?

Git is a distributed version control system that allows you to track changes to files and collaborate with others on projects. It helps manage different versions of your code, revert to previous versions if needed, create branches to work on features separately, and merge changes back together. Git is widely used by developers to manage source code, but it can track changes to any set of files.

Getting Started With Git

Git is fairly straightforward to install on your computer.

  1. Visit the official Git website (https://git-scm.com) and find the "Downloads" section.
  2. Find your operating system ("Windows", "Mac", or "Linux") and click on the appropriate link.
  3. Follow all the installation instructions.
  4. To verify that everything was installed correctly, open up the terminal or command line and type git version

Connecting Git to Github

GitHub is a web-based platform that provides hosting for Git repositories and facilitates collaboration. If you're planning on collaborating with other devs or sharing your projects with the world, then you need to connect your local projects to a GitHub account. To connect your Git repositories to GitHub, follow these steps:

  1. Create a GitHub account if you don't already have one. Visit GitHub's website and sign up.
  2. Create a new repository on GitHub. Click the "+" icon in the top-right corner and select "New repository". Give your repository a name, choose the visibility (public or private), and click "Create repository".
  3. Copy the URL of your GitHub repository. On the repository page, click the "Code" button and copy the HTTPS or SSH URL.
  4. Open a terminal or command prompt and navigate to your local Git repository.
  5. Add the GitHub repository as a remote by typing in the following:
    • git remote add origin <repository-url>
    • Note: Replace <repository-url> with the URL you copied from GitHub.
  6. Verify the remote was added successfully by typing: git remote -v . You should see the URL of your GitHub repository listed.
  7. Once you are ready to push your local commits to GitHub, type the following:
    • git push -u origin main
    This pushes your commits to the `main` branch on GitHub. If your default branch has a different name, replace `main` with the appropriate branch name.

Now your local Git repository is connected to GitHub! You can push commits, collaborate with others, and take advantage of GitHub's features.

Common Commands in Git

Now that you are all set up, it's time to use Git to manage your code and projects. Below are some of the most common commands you will use in Git for your version control.

  1. git init: Initializes a new Git repository in the current directory.
  2. git clone <url>: Creates a copy of a Git repository from the specified URL.
  3. git add <file>: Adds changes in a file to the staging area to be committed. Use `[git add .]` to add all changes.
  4. git commit -m "message": Commits the staged changes with the specified message describing the changes.
  5. git status: Shows the current status of the repository, including staged, unstaged, and untracked files.
  6. git branch: Lists all the branches in the repository. Use `git branch <branch-name< to create a new branch.
  7. git checkout <branch-name>: Switches to the specified branch. Use git checkout -b <branch-name> to create a new branch and switch to it.
  8. git merge <branch-name>: Merges the specified branch into the current branch.
  9. git pull: Fetches changes from the remote repository and merges them into the current branch.
  10. git push: Pushes committed changes to the remote repository.

Still need help?

If you're still having problems or have questions about using Git, don't worry! There are plenty of resources available to help you:

  1. Official Git Documentation: The official Git documentation is a comprehensive resource that covers all aspects of Git. It includes a reference manual, a book on Git pro, and guides for various Git commands and concepts.
  2. GitHub Guides: GitHub provides a set of guides that cover various topics related to Git and GitHub. These guides are beginner-friendly and provide step-by-step instructions for common tasks.
  3. Stack Overflow: Stack Overflow is a popular Q&A platform where developers can ask questions and find answers. You can search for Git-related questions or post your own question to get help from the community.
  4. Git Forums and Communities: There are several online forums and communities dedicated to Git where you can ask questions, share experiences, and learn from others. Some popular ones include:
    • Git Community Book
    • Git User's Manual Mailing List
    • Stack Overflow Git Tag

Remember, learning Git takes time and practice. Don't hesitate to experiment, make mistakes, and seek help when needed. With persistence and the right resources, you'll become proficient in using Git for version control and collaboration.
